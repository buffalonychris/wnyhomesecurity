# WNYHS QR Funnel Standards — REV01

Status: LOCKED (governance standard)
Context: CTX-WNYHS-FINISH-LINE-REV01

## 1. QR Funnel Purpose

The QR funnel is:
- focused public placard scan intake
- high-conversion local estimate request
- not a full website squeezed into one page

## 2. QR Navigation Standard

QR nav must include:
1. View Packages
2. See Our Work
3. Call/Text
4. Schedule Estimate

## 3. QR Form Protection Rules

Do not:
- change HubSpot/CRM payload
- remove/rename form fields
- change source tracking
- change consent payloads

Allowed:
- visual layout updates only

## 4. Preferred Progressive Layout

Preferred stages:
1. Stage 1: name, phone/email, what they want protected
2. Stage 2: address/property details
3. Stage 3: estimate window
4. Stage 4: contact permissions

## 5. Consent Structure Rule

Consent UX must clearly separate:
- preferred contact method
- permission to contact

## 6. Must-Remove Content Leakage

Remove/avoid:
- assistant references
- SaaS references
- demo references
- Operator footer
- Start Free Demo / View Pricing / Become a Partner leakage

## 7. Trust Structure Requirements

Trust messaging should emphasize:
- local installation
- equipment options you can own
- cameras/sensors/alarms/automations
- no national call center

## 8. Protected QR Runtime Boundaries

Protected and unchanged:
- HubSpot
- lead API
- QR source tracking
- form submission behavior
