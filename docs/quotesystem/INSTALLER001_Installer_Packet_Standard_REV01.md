# INSTALLER001 Installer Packet Standard REV01

Status: ACTIVE
Customer-facing: No
Implementation authority: No

## Purpose

This standard defines the quote-system installer packet governance for WNYHS install readiness.

It is documentation only. It does not authorize installer software, scheduling implementation, route/UI changes, inventory implementation, quote generation, payment changes, or runtime behavior changes.

## Onsite Time Standard

All jobs should be scoped to complete within an 8-hour onsite window.

Target working time is 6 hours.

Scope that cannot reasonably fit the onsite time standard should be flagged before scheduling and before installer packet approval.

## Task Ownership Buckets

Installer packet tasks should be assigned to:

- Installer A.
- Installer B.
- Either installer.
- Both installers required.

Current defaults:

- Installer A = Chris.
- Installer B = Lou.

## Required Installer Packet Contents

The installer packet must include:

- Part pick list.
- Hardware-by-location list.
- Floorplan placement references.
- Install steps.
- Mounting notes.
- Test procedures.
- Home Assistant/dashboard prep notes.
- Customer handoff steps.
- Exceptions/uncompleted item log.

## Warehouse Departure Gate

No installer should leave the warehouse without the parts and job packet required for assigned tasks.

The packet should align with the approved Property Model, BOM, inventory reservation, and installer task ownership.

## Exception Log

Any exception or uncompleted item should be documented in the installer packet before customer handoff is treated as complete.

## Boundary

This standard defines installer packet content and readiness gates only. It does not implement job scheduling, inventory software, technician routing, time tracking, customer signatures, payment collection, or Home Assistant dashboard generation.
