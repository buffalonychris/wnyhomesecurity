# WNY Home Security Visual System — REV01

Status: Active for VISUAL-SYSTEM001 under FINISH-LINE-PAGES001.

## 1) Typography Hierarchy
- Use condensed display typography only for primary section headlines and short impact lines.
- Use readable sans body/support type for all supporting copy, lists, captions, and trust/detail text.
- Max desktop headline sizing: `clamp(2.1rem, 4vw, 3.2rem)` for hero H1 and `clamp(1.6rem, 3vw, 2.3rem)` for section H2.
- Mobile headline behavior: reduce line density, target 2–4 lines max, and keep CTA visible without excessive scrolling.
- Do not apply aggressive display styling to body paragraphs or repeated card titles.

## 2) Logo Usage Rules
- Crest/shield is the authority mark and should be used as a subtle trust signal.
- Wide stripe/banner logo is a supporting lockup; it should not dominate hero composition.
- Avoid large standalone logo blocks that resemble ad units.
- Logo placement must reinforce hierarchy after headline + subhead + CTA.

## 3) Hero Composition Rules
- One dominant message, one dominant visual anchor, one clear CTA row.
- Remove oversized dead zones and avoid decorative blocks that interrupt reading flow.
- Use crest as atmospheric authority (small badge/watermark), not as giant filler.
- Keep headline, proof, and action path in one coherent scan pattern.

## 4) Image Treatment Rules
- Use premium card surfaces with consistent media framing.
- Keep gallery aspect ratio standards (`16:10` for feature media, `4:3` for supporting cards).
- Apply restrained shadows/glow; avoid heavy effects.
- Avoid generic repeated grid dumps; curate sections by use-case and priority.

## 5) Spacing Rhythm
- Standard content max-widths: hero/content `~1080px`, dense copy panels `~820px`.
- Vertical rhythm baseline: 24–32px internal gaps, 56–80px section spacing.
- Group related content in cards/panels to reduce visual noise.
- Avoid oversized empty black regions with no information value.

## 6) Button Sizing Consistency
- Primary CTA: visually dominant but restrained (44–50px effective height, medium-large padding).
- Secondary CTA: equal alignment with lower visual weight.
- Avoid oversized “cartoon” buttons and inconsistent CTA heights.
- On mobile, CTA rows stack with full-width behavior where horizontal space is constrained.

## 7) Card Depth + Section Surfaces
- Use dark graphite panels with subtle border separation.
- Keep restrained gold accents for emphasis and state.
- Preserve semantic tokens; do not introduce random ad-hoc colors.
- Avoid heavy cyan/blue dominance unless already defined by semantic tokens.

## 8) Section Widths
- Homepage hero body: medium-wide for fast scan (`~1080px` cap).
- Content panels: medium width (`~980px` cap).
- Gallery grids: variable responsive columns with consistent card spacing.
- CTA blocks: centered and tighter width to keep focus.

## 9) Premium Motion Constraints
- Motion must be subtle, optional, and non-gimmicky.
- No flashy transitions or attention-hijacking animation loops.
- Respect reduced-motion preferences and keep transforms minimal.

## 10) Visual Density Targets
- Favor fewer, stronger elements instead of many weak elements.
- Remove repeated low-value card repetition.
- Prefer curated story sections over brochure-like dumps.
- Make each major section support conversion progression.

## 11) Dark-Mode Tonal Balance
- Base: near-black background.
- Surfaces: graphite/lifted dark panels.
- Text: white/off-white for primary + softer neutrals for support text.
- Accent: restrained warm-gold highlights.
- Avoid washed-out blue-tinted UI tone.

## 12) Conversion-Oriented Structure
Recommended order on key pages:
1. Headline (clear offer)
2. Proof (what makes it credible)
3. CTA (next action)
4. Trust/safety context
5. Visual evidence
6. Next-step action block
