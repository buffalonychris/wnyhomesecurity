## 1. Purpose
REV03 applies the Deep Research validation findings to the already-merged REV02 capability universe. It preserves the original 187 feature rows and adds validation-oriented planning fields for Home Assistant integration, local-only viability, complexity, market interest, dependencies, launch suitability, BOM priority, and unresolved notes.

REV03 is not a public pricing document, BOM document, package catalog, customer-facing claims document, or final hardware approval document. It does not approve any feature for customer deployment without follow-on bench validation, BOM work, and claims review where applicable.

- `docs/catalogs/wnyhs_capability_catalog_rev02.csv`

## 3. Governing Principle
WNYHS validates customer outcomes before packaging them. A capability should move from catalog row to bench validation, then BOM research, then package planning, then customer-facing language only after the technical path and claims boundaries are understood.

## 4. WNYHS Core Positioning
WNYHS Core remains the internal planning anchor for local control, customer ownership, privacy, expansion capability, dashboards, automations, and device-family replaceability. Device names in this document are candidate validation references only and are not final hardware commitments.

## 5. Local-First Device Selection Rule
Prefer local Home Assistant paths, Z-Wave, Zigbee, PoE, MQTT, ESPHome, local NVR, local polling, local push, and documented local-network integrations. If normal use depends on vendor apps, remote push infrastructure, cloud accounts, subscriptions, custom code, HACS maintenance, or uncertain device behavior, classify the row conservatively.
