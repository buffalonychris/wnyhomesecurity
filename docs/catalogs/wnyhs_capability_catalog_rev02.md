# WNYHS Capability Catalog REV02 Baseline

Version: REV02
Status: Planning catalog / source-of-truth candidate
Audience: Internal planning only
Customer-facing: No
Public pricing approved: No

## 1. Purpose

This catalog defines the WNYHS control-plane capability universe for future planning. It is used to support Home Assistant integration validation, local-only viability validation, future BOM research, solution package design, WNYHS Core page planning, website expansion, future customer-facing package copy, installation standards, and approved hardware standards.

This document is not a customer-facing package document, not a final hardware commitment document, and not a public offer. It does not approve any feature for customer deployment without follow-on validation.

## 2. Governing Principle

WNYHS plans around customer outcomes first, then validates feasibility, then builds BOMs, then groups validated capabilities into solution packages, then evaluates pricing after the underlying BOMs and deployment standards are known.

Devices are implementation details. Vendor hardware should remain replaceable where practical, and no capability should depend on a vendor app, vendor cloud, required subscription, or third-party customer app access for normal operation unless the exception is explicitly documented and accepted in a later revision.

## 3. WNYHS Core Positioning

WNYHS Core is the intended local control plane behind every solution. It is the planning anchor for ownership, privacy, local control, expansion capability, and professionally installed smart-property functionality.

A capability may require sensors, cameras, relays, network equipment, dashboards, automations, or notifications, but the durable planning model is the WNYHS Core environment plus replaceable device families, not a single vendor device list.

## 4. Local-First Device Selection Rule

Prefer candidate hardware families and integration paths that support local control through WNYHS Core, Home Assistant, Z-Wave, Zigbee, PoE, MQTT, Node-RED, or local-network integrations. If a capability has cloud dependency, vendor-app dependency, subscription dependency, or uncertain local-only viability, mark it conservatively and require REV03 research before package or BOM approval.

- Validate complexity and install risks against real WNY property conditions, including network coverage, detached structures, winter conditions, and equipment-room access.
