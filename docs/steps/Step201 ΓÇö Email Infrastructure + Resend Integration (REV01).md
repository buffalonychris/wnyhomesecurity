## 5. Email Architecture

Inbound email remains handled by Cloudflare Email Routing.

```txt
Customer → Cloudflare Email Routing → Gmail / Operators
```

Outbound system-generated email is handled by Resend.

```txt
Website / Backend → Resend → Customer / Operators
```

Resend is for sending only under this step.

Do not enable or implement Resend receiving on the root domain.

## 7. Mandatory Audit Copy Rule

Every email generated by the site must send an internal copy to WNY Home Security.

This applies to:

- Customer-facing quote emails
- Customer-facing agreement emails
- Customer-facing support or response emails
- Contact/intake form emails
- Schedule/install request emails
- Internal notification emails
- Any other outbound email generated by the site

Preferred implementation:

- Use BCC for customer-facing emails when appropriate.
- Use explicit internal notification emails when BCC is not supported or not appropriate.

Do not rely only on logs.

Do not silently skip audit copies.

If audit delivery cannot be confirmed, treat the send as failed unless the implementation proves the customer email and audit copy are sent in one accepted Resend API call.

MAIL_SUPPORT_TO=

MAIL_INSTALL_TO=
MAIL_ADMIN_TO=

Rules:

- Do not expose `RESEND_API_KEY` client-side.
- Do not commit secrets.
- Do not hardcode private recipient addresses in frontend code.
- Public role addresses may be referenced as customer-facing reply-to values when appropriate.

## 10. Implementation Requirements

Implementation must:

- Use a centralized Resend helper.
- Support `to`, `cc`, `bcc`, `replyTo`, `subject`, `html`, and `text` where applicable.
- Run server-side only.
- Remain compatible with Cloudflare Pages Functions or the repo’s existing serverless structure.
- Preserve existing routes and payload structures.
- Preserve existing quote/agreement/payment behavior.
- Include plain-text fallback where reasonable.
- Prefix operational subjects with `[WNYHS]`.

---

## 11. Error Handling Requirements

If Resend fails:

- Show a user-safe error.
- Do not expose stack traces.
- Log enough server-side detail for debugging.
- Preserve form data where practical.
- Do not falsely claim email was sent.

If customer email is missing:

- Route internally only.
- Make the subject clear that the customer email was not provided.

If audit copy fails:

- Treat as failure unless customer and audit delivery are part of the same accepted send operation.
