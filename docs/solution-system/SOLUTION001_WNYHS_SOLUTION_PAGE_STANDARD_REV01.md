# SOLUTION001_WNYHS_SOLUTION_PAGE_STANDARD_REV01

## Purpose

This document establishes the first formal WNY Home Security Solution Page Standard.

The purpose of this standard is to eliminate page drift as the WNY Home Security solution catalog expands.

This standard governs:

* Solution page structure
* Visual hierarchy
* Typography
* Card layout
* CTA standards
* Copy standards
* Image standards
* Internal linking standards

This revision is intended to standardize and improve:

* /solutions/senior-safety
* /solutions/water-protection
* /solutions/family-awareness
* /solutions/vacation-homes

Following visual approval, this standard may be promoted to a locked governance document.

---

# Core Principle

Every solution page must answer:

"Can WNY Home Security help solve this homeowner problem in a practical, local, and low-pressure way?"

Solution pages are not:

* Hardware catalogs
* Generic service pages
* Company overview pages
* Trust-heavy marketing pages

Solution pages are:

* Problem-focused
* Outcome-oriented
* Educational
* Conversion-supportive

---

# Page Responsibilities

* Why should I care?

* Solution selection
* Starting-point comparison
* Pricing orientation

* What are my options?
* What do they cost?

## Solution Pages

Purpose:

* Problem resolution

* Educational conversion

# Required Solution Page Structure

All future solution pages must follow this structure.

## Section 1 — Hero

Required:

* Eyebrow
* Problem-focused H1
* Supporting paragraph
* Trust pill row
* Primary CTA
* Secondary CTA
* Optional hero image

Trust pills:

* Local Smart Property Solutions
* No Required Monthly Fees
* Customer-Owned Equipment
* Professionally Installed & Locally Supported

---

## Section 2 — Who This Helps

Purpose:

Allow the visitor to self-identify.

Examples:

* Adult children supporting aging parents
* Homeowners worried about frozen pipes
* Families managing arrivals and packages
* Seasonal property owners

Focus on people and situations.

Not hardware.

---

## Section 3 — Problem / Solution / Outcome

Required:

Three-column layout.

### Problem

Describe concern.

### Solution

Describe planning approach.

### Outcome

Describe awareness, confidence, and practical control.

Avoid guarantees.

Use:

* Helps you know sooner
* Helps reduce uncertainty
* Supports better decisions

---

## Section 4 — Western New York Context

Purpose:

Demonstrate local relevance.

Include:

* WNY winters
* Spring thaw
* Older homes
* Basements
* Detached garages
* Seasonal properties
* Family routines

Avoid fabricated local stories.

---

## Section 5 — Possible Pieces Of The Plan

Purpose:

Illustrate possible solution components.

Format:

* Pills
* Tiles
* Short labels

Examples:

Water Protection:

* Water Sensor Placement
* Temperature Awareness
* Sump Awareness
* Mechanical Room Planning
* Alert Routing

Family Awareness:

* Door Awareness
* Garage Awareness
* Package Visibility
* Driveway Visibility
* Smart Lock Planning

Do not imply fixed scope.

Do not imply pricing.

---

## Section 6 — Example Scenario

Purpose:

Add emotional relevance without inventing customer stories.

Required.

Allowed labels:

* Example Scenario
* Common Situation
* Typical Situation
* How This Might Look

Do not:

* Use customer names
* Invent towns
* Invent outcomes

---

## Section 7 — Next Step

Purpose:

Convert without pressure.

Preferred CTA hierarchy:

1. Request Estimate
2. Start Fit Check
3. Talk Through Your Property
4. Compare Starting Points
5. See Example Work

Keep CTA language consistent.

---

# Sections Not Required

## Why WNY Home Security

Not required on individual solution pages.

Reason:

Trust-building primarily occurs on:

* Homepage
* Packages
* Our Work

Solution pages should remain focused on the homeowner problem.

Trust reinforcement should remain limited to:

* Hero trust pills
* Local references
* Ownership language

---

# Visual Standard

Homepage is the canonical design reference.

Solution pages should visually align with:

* Homepage
* Packages
* QR Landing

Use existing:

* Semantic tokens
* Borders
* Radius values
* Shadows
* Card styles
* CTA styles

Do not create a separate design language.

---

# Typography Standard

## Eyebrows

Use homepage styling.

Requirements:

* Small
* Uppercase
* Accent color
* Consistent spacing

---

## H1

Requirements:

* Large
* Problem-focused
* Same hierarchy as homepage hero

---

## H2

Requirements:

* Same hierarchy as homepage section titles
* Concise
* Clear

---

## Body Copy

Requirements:

* Homeowner language
* Short paragraphs
* Minimal jargon

---

# Card Standard

Cards should match homepage styling.

Requirements:

* Consistent border treatment
* Consistent radius
* Consistent padding
* Consistent spacing
* Equal-height layouts where practical

No page-specific card designs.

---

# Image Standard

AI-generated imagery is allowed.

Requirements:

* Illustrative only
* No fake customer installations
* No fake before/after claims
* No panic-based imagery

Preferred concepts:

Senior Safety:

* Calm home environment
* Independence
* Entry awareness

Water Protection:

* Utility room
* Basement
* Winter context

Family Awareness:

* Entry
* Garage
* Driveway
* Package visibility

Vacation Homes:

* Seasonal home
* Cabin
* Winter property

Maximum:

* One hero image
* One supporting image

---

# CTA Standard

Primary:

Request Estimate

Secondary:

Start Fit Check

Optional:

* Talk Through Your Property
* Compare Starting Points
* See Example Work

Avoid:

* Buy Now
* Protect Me Today
* High-pressure language

---

# Copy Guardrails

Allowed:

* Helps you know sooner
* Practical awareness
* Customer-owned equipment
* No required monthly fees
* Local smart property solutions
* Built around your property

Forbidden:

* Guaranteed protection
* Guaranteed prevention
* Guaranteed detection
* Medical monitoring
* Emergency response claims
* Insurance savings claims
* Fake customer stories
* Fake testimonials
* Fake statistics

---

# Internal Linking Standard

Required:

* Packages
* Fit Check
* Estimate

Optional:

* Our Work
* Support

Avoid excessive linking.

---

# SOLUTION001-A Scope

Purpose:

Update the four existing solution pages to conform to this standard.

Allowed:

* Solution page components
* Solution page content
* Solution page styling
* Site version bump
* Documentation updates

Not Allowed:

* Homepage changes
* Packages changes
* Support changes
* Our Work changes
* HubSpot changes
* Stripe changes
* Scheduling changes
* Environment changes

---

# Approval Workflow

1. Add this document to repo.
2. Execute SOLUTION001-A.
3. Visually review all four pages.
4. Approve structure and styling.
5. Create SOLUTION001-B.
6. Lock standard.

Future solution pages must follow the locked standard.

---

# Success Criteria

All solution pages should:

* Feel like part of the same system
* Follow the same structure
* Follow the same typography
* Follow the same CTA hierarchy
* Maintain local relevance
* Remain problem-first
* Avoid unsupported claims

The goal is not more pages.

The goal is a repeatable WNY Home Security solution-page system.
